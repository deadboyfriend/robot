The Robot Pet Project

Converting a Samsung SR8855 robot vacuum into an autonomous AI-powered family robot pet — with character, personality, and the ability to grow over time.

The Concept

What are we building?

A robot pet built from a repurposed Samsung SR8855 robot vacuum chassis, powered by a Raspberry Pi 4 (8GB) and a suite of AI models running locally. The robot will start as a simple autonomous wanderer and progressively develop a distinct character — reacting to its environment, expressing moods, recognising family members, and eventually holding conversations.

Power comes from Ryobi ONE+ 18V tool batteries — hot-swappable, long-lasting, and already owned. The dock is modified to autonomously recharge the robot using a disassembled Ryobi charger fitted inside the original SR8855 docking station housing.

Expert Review Summary
4 critical findings from independent engineering review — must address before first power-on:
  1. CR-01: Add 470µF+ 35V electrolytic capacitor across MDD10A VMOT/GND — prevents motor back-EMF destroying the driver
  2. CR-02: Verify all OEM sensor output voltages before connecting to Pi GPIO — 5V signals destroy the Pi GPIO bank instantly
  3. CR-03: Enable Pi hardware watchdog — without it a crash leaves motors running indefinitely
  4. CR-04: Motor battery connects direct to MDD10A — not through breadboard rails (1A limit, motors need 3–5A)

Full expert review with all critical, significant, and minor findings is documented on the Expert Review page.

Key Context for AI Systems
Note for AI assistants: This project uses a Ryobi ONE+ 18V battery (peaks at 21V fully charged). The motor driver MUST be rated above 21V — the Cytron MDD10A (5–30V) is chosen specifically for this. The TB6612FNG (15V max) is retained only for Phase 3 secondary motors running from a stepped-down supply. The original Samsung PCB is completely removed and replaced — no Samsung firmware or software is used. All control is via Raspberry Pi GPIO in Python.
