Wiring Guide

Step-by-step wiring for Phase 1 bench testing. Start on breadboard with AA batteries and a test motor — validate the circuit before touching the vacuum chassis.

Before You Start
Breadboard-first rule: Build and test the entire circuit on a breadboard with a 4×AA battery pack and a small spare DC motor. Only when the Python test script drives the motor correctly should you progress to connecting the vacuum's motors and Ryobi battery. This prevents expensive mistakes on the real chassis.

Safety checklist

  • Measure battery voltage before connecting anything (4×AA = ~6V, Ryobi = 18–21V)
  • Verify MDD10A polarity with multimeter — no reverse protection
  • Pi powered OFF before wiring GPIO connections
  • Check GPIO voltage is 3.3V (not 5V) before connecting to MDD10A logic
  • Double-check all connections against table below before powering on

Test with LED first

Replace the motor and battery with a single LED (any colour) and 330Ω resistor between AO1 and GND. When the Python test script runs, the LED flickers with PWM. This confirms all connections are correct without motor/battery risk. Once working — swap in the real motor.

Phase 1 Bench Wiring — Pi 4 + Cytron MDD10A
Note on motor driver change: The bench wiring diagram was originally designed for the TB6612FNG. The Cytron MDD10A is used instead (due to Ryobi 18V compatibility). Pinout is different — use the MDD10A connection table below, not any TB6612FNG diagrams.

T-cobbler tip: The T-cobbler breakout uses BCM numbering — e.g. "GPIO 18" on the cobbler = BCM 18 = physical board pin 12. Always reference the BCM number in Python code (when using GPIO.setmode(GPIO.BCM)). The physical pin numbers in the table above are just for locating pins on the Pi header.
Sensor Wiring — SR8855 OEM Sensors
Assess before buying replacements: The SR8855's cliff sensors and obstacle sensors may be directly usable with the Pi. During teardown, photograph the sensor connector types and wire colours, then probe the output with a multimeter to determine: signal voltage (3.3V or 5V), output type (digital HIGH/LOW or analog), and number of wires (typically 3: VCC, GND, OUT). 3.3V digital output = plug straight into Pi GPIO. 5V output = needs a voltage divider or level shifter.

SymptomLikely causeFix
Motor doesn't spin at allVMOT not connected, or battery not providing powerMeasure VMOT with multimeter — should read battery voltage when powered
Motor spins wrong directionMotor polarity reversedSwap A+ and A− at the motor terminals. Or flip DIR pin logic in code.
Motor runs but Pi can't control itGround not shared between Pi and motor circuitEnsure Pi GND and battery GND are on the same breadboard rail
Pi boots but GPIO does nothingWrong GPIO mode or pin numbersVerify BCM mode set in code. Check physical pin vs BCM numbering.
Erratic motor behaviourPower supply can't supply enough currentCheck battery voltage under load. 4×AA may sag. Try fresh batteries.
