Step-by-step checklist

  • Step 1: Set up Pi 4 with Raspberry Pi OS 64-bit Lite. Enable SSH.
  • Step 2: Build bench circuit on breadboard — Cytron MDD10A + 4×AA batteries + test motor. No vacuum yet.
  • Step 3: Run LED test (LED + 330Ω resistor instead of motor). Verify GPIO control works.
  • Step 4: Run motor test script — forward, reverse, turn. Verify motor responds correctly.
  • Step 5: Open SR8855 chassis (see teardown guide). Document all connectors and sensors.
  • Step 6: Identify and probe cliff sensor and bump sensor outputs with multimeter.
  • Step 7: Connect cliff sensors to Pi GPIO. Test with Python — verify HIGH/LOW readings.
  • Step 8: Integrate Ryobi battery adapter. Verify buck converter outputs 5V before connecting Pi.
  • Step 9: Connect drive motors to MDD10A. Test forward/reverse/turn on bench.
  • Step 10: Mount Pi, MDD10A, and buck converter inside chassis. Route all wiring.
  • Step 11: Run wandering script with robot on floor. Observe and tune behaviour.

Phase 4 — The Brain

Let it think, remember, and talk back

Planned
Overall progress0%

What you build

  • USB microphone + Faster-Whisper STT on Pi
  • Wake word detection ("Hey [robot name]!")
  • Ollama LLM on home PC (Phi-3 Mini or Llama 3.2 3B)
  • Pi calls Ollama over WiFi — fast enough (<2s response)
  • Piper TTS — synthesised robot voice on Pi
  • Persistent memory.json — names, preferences, events
  • Family face recognition (from Phase 2 camera)

What it can do

  • Hold conversations in a consistent character voice
  • Remember family names, facts, and preferences across sessions
  • Respond differently to each family member
  • Develop in-jokes and running stories
  • Announce when battery is low ("I'm hungry!")
  • Report what it's been doing ("I explored the kitchen today")
Key architecture decision: The LLM does NOT run on the Pi 4 — it's too slow. Ollama runs on any capable home PC on the same WiFi network. The Pi handles audio I/O, sends text over HTTP to Ollama, and speaks the response with Piper TTS. Response latency is typically under 2 seconds on a mid-range PC, which feels natural for a robot pet.
